# MySQL 索引优化实践

## 索引类型
- **B+Tree 索引** — 最常用，适合范围查询
- **Hash 索引** — 等值查询快，不支持范围
- **全文索引** — 用于文本搜索

## 优化原则
1. 最左前缀原则：联合索引 (a, b, c) 可覆盖 a / a,b / a,b,c 查询
2. 覆盖索引：查询字段全在索引中，避免回表
3. 避免索引失效：
   - 对索引列使用函数
   - 隐式类型转换
   - LIKE '%xx' 前模糊

## EXPLAIN 关键字段
| 字段 | 关注点 |
|------|--------|
| type | 至少 ref，避免 ALL |
| key | 实际使用的索引 |
| rows | 扫描行数，越小越好 |
| Extra | Using index 最优 |
