# 密钥与敏感信息管理

## 原则
- **永远不把密钥提交到代码仓库**
- 使用环境变量或密钥管理服务
- 定期轮换密钥

## 方案对比
| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| 环境变量 | 小型项目 | 简单 | 不易管理 |
| .env 文件 | 本地开发 | 方便 | 需 gitignore |
| Vault (HashiCorp) | 企业级 | 功能强大 | 运维复杂 |
| 云 KMS | 云上服务 | 托管免运维 | 厂商绑定 |

## 最佳实践
1. `.env` 加入 `.gitignore`
2. 提供 `.env.example` 作为模板
3. CI/CD 中使用 Secret 变量
4. 生产环境使用 KMS 或 Vault
5. 审计日志记录密钥访问
