# Redis 缓存策略

## 缓存模式
### Cache Aside（旁路缓存）
1. 读：先查缓存 → 未命中则查 DB → 写入缓存
2. 写：更新 DB → 删除缓存

### Write Through（写穿透）
写入时同时更新缓存和 DB，由缓存层保证一致性。

### Write Behind（异步写回）
写入只更新缓存，异步批量刷入 DB。高性能但有数据丢失风险。

## 常见问题及解决
| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 缓存穿透 | 查询不存在的 key | 布隆过滤器 / 空值缓存 |
| 缓存击穿 | 热点 key 过期瞬间 | 互斥锁 / 永不过期+异步更新 |
| 缓存雪崩 | 大量 key 同时过期 | 随机过期时间 / 多级缓存 |

## TTL 设置建议
- 用户信息：30 分钟
- 配置数据：5 分钟
- 排行榜：10 秒
